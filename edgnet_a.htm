EDGARsuite™ - Network Administrator's Reference

Setting up EDGARsuite™ for network operation

  1. Run the EDGARsuite™ installer on the file server from the server console (do not do this remotely).

  2. Create a folder named ES.SYS within the folder where EDGARsuite™ is installed (typically, C:\Aci Programs\EDGsuite).

  3. Assign each EDGARsuite™ user a unique User ID (up to 8 letters and/or numbers). For each user, create a folder within the EDGARsuite folder, named x.USR (where x is the User ID). To remove a user permanently, remove the corresponding x.USR folder. A user may be added at any time, but removed only when he/she is not using the software.

Setting up Client computers

To set up EDGARsuite™ on a client computer, run the file EdgIns.exe located in the folder where EDGARsuite™ is installed. This file must be run on each client computer from its original location on the server. Do not copy the file to the client computer. Beyond this, the installation process is automatic. It creates a folder named EDGARsuite on the client's desktop and in the Start Menu with links to software and resources on the server.

User Management

To identify users who are using EDGARsuite™, look at the contents of the ES.SYS folder. Each active User ID using the program is shown there by a file bearing the name of the user ID, with .ON extension. The date and time of this file indicates when that copy of EDGARsuite™ was started.

EDGARsuite™ adds a UserID.ON file to the ES.SYS folder when a user starts the program, and deletes it on exit. However, if the program terminates abnormally, the UserID.ON file may remain there, and must be deleted manually. When doing so, first ensure that the user is not using the program. Never delete a UserID.ON file while a user is using the program.

Security and Advanced Options

For added network security, you may write-protect the folder where EDGARsuite™ is installed (typically, C:\Aci Programs\EDGsuite). The ES.SYS folder must, however, support read+write access. If you wish, you may assign individual users access only to their own specific UserID.USR folders to keep them from using other users' accounts.

For advanced networking options, you may create a text file called ES.NET in the folder where EDGARsuite™ is installed, into which you may place one or more of the following commands:

SYS=SysDirectory
USR=UsrDirectory
CUST=Yes/No
DRIVES=Yes/No

where:

SysDirectory represents the complete path name of the folder which should replace the ES.SYS folder. If this entry is not specified, ES.SYS as described above is used.

UsrDirectory represents the complete path name template of the directories to replace the UserID.USR folders. In this template, an asterisk (*) may be used as a place holder for the User ID. If this entry is not specified, the UserID.USR folders as described above are used.

CUST= specifies whether or not users should be allowed access to EDGARwiz's Advanced Customization options. If this entry is not specified, Yes is assumed. This entry does not apply to the other EDGARsuite™ components.

DRIVES= specifies whether or not network drives should show up in users' file navigation dialog boxes. If this entry is not specified, Yes is assumed.

The above entries may be entered in upper, lower or mixed case. There should be no spaces within an entry, not even before or after the '=' symbol.
